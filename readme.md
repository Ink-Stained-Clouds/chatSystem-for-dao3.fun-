# Readme
/\*

  

变量命名规范

使用小驼峰命名法 (camelCase)

  

函数命名规范

使用小驼峰命名法 (camelCase)

  

类命名规范

使用大驼峰命名法 (PascalCase)

  

常量命名规范

使用全大写字母，单词间用下划线分隔 (SCREAMING\_SNAKE\_CASE)

  

文件名命名规范

使用小驼峰命名法 (camelCase) 或短横线命名法 (kebab-case)

  

模块/导出命名规范

默认导出: 使用大驼峰命名法 (PascalCase) 或小驼峰命名法 (camelCase)

命名导出: 使用大驼峰命名法 (PascalCase) 或小驼峰命名法 (camelCase)

模块名应与文件名保持一致

  

参数命名规范

使用小驼峰命名法 (camelCase)

  

私有成员命名规范

使用小驼峰命名法 (camelCase) 前缀加下划线 (\_)

  

事件处理函数命名规范

使用小驼峰命名法 (camelCase) 前缀加 "handle"

  

状态变量命名规范

使用小驼峰命名法 (camelCase)

  

布尔值命名规范

使用小驼峰命名法 (camelCase) 前缀加 "is", "has", "can", "should"

  

数组命名规范

使用小驼峰命名法 (camelCase)，使用复数形式

  

对象命名规范

使用小驼峰命名法 (camelCase)，使用单数形式

  

类型命名规范

使用大驼峰命名法 (PascalCase)

  

枚举命名规范

使用大驼峰命名法 (PascalCase) 枚举值使用全大写

  

接口命名规范

使用大驼峰命名法 (PascalCase) 前缀加 "I"

  

错误处理命名规范

使用小驼峰命名法 (camelCase) 前缀加 "error"

  

测试相关命名规范

测试文件名: 使用小驼峰命名法 (camelCase) 子后缀 ".test" 或 ".spec"

测试函数名: 使用小驼峰命名法 (camelCase) 前缀加 "test"

  

注释规范

除 \[标记注释, 必要的JSDoc, 数据值解释注释\] 外, 本代码不使用任何注释
